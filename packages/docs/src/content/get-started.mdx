# Getting Started

This guide will help you set up and start using the Rhinolabs UI Toolkit in your React application.

## Prerequisites

- Node.js 16.8 or later
- React 18 or later
- TypeScript 4.5 or later (recommended)

## Installation

You can install the libraries individually based on your needs:

```bash
# Using npm
npm install @rhinolabs/ui @rhinolabs/hooks

# Using pnpm
pnpm add @rhinolabs/ui @rhinolabs/hooks

# Using yarn
yarn add @rhinolabs/ui @rhinolabs/hooks
```

## Usage

### Using UI Components

```tsx
import { Button } from "@rhinolabs/ui";

function App() {
  return (
    <Button variant="default">
      Click me
    </Button>
  );
}
```

### Component Patterns

Many of our components use the compound pattern, which provides a more intuitive and maintainable way to compose complex components. Instead of importing multiple components separately, you can access subcomponents through the main component using dot notation.

For example, instead of:

```tsx
import { Card, CardHeader, CardTitle, CardDescription, CardContent } from "@rhinolabs/ui";

function App() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>Title</CardTitle>
        <CardDescription>Description</CardDescription>
      </CardHeader>
      <CardContent>Content</CardContent>
    </Card>
  );
}
```

You can write:

```tsx
import { Card } from "@rhinolabs/ui";

function App() {
  return (
    <Card>
      <Card.Header>
        <Card.Title>Title</Card.Title>
        <Card.Description>Description</Card.Description>
      </Card.Header>
      <Card.Content>Content</Card.Content>
    </Card>
  );
}
```

This pattern is used across many components including:
- Alert (Alert.Title, Alert.Description)
- Breadcrumb (Breadcrumb.List, Breadcrumb.Item, etc.)
- Card (Card.Header, Card.Title, etc.)
- Collapsible (Collapsible.Trigger, Collapsible.Content)
- Command (Command.Input, Command.List, etc.)
- ContextMenu (ContextMenu.Trigger, ContextMenu.Content, etc.)
- Dialog (Dialog.Trigger, Dialog.Content, etc.)
- DropdownMenu (DropdownMenu.Trigger, DropdownMenu.Content, etc.)
- Form (Form.Field, Form.Label, etc.)
- Popover (Popover.Trigger, Popover.Content, etc.)
- RadioGroup (RadioGroup.Item)
- ResizablePanelGroup (ResizablePanelGroup.Panel, ResizablePanelGroup.Handle)
- ScrollArea (ScrollArea.Bar)
- Select (Select.Trigger, Select.Content, etc.)
- Sheet (Sheet.Trigger, Sheet.Content, etc.)
- Sidebar (Sidebar.Content, Sidebar.Header, etc.)
- Table (Table.Header, Table.Body, etc.)
- Tabs (Tabs.List, Tabs.Trigger, etc.)
- ToggleGroup (ToggleGroup.Item)
- Tooltip (Tooltip.Trigger, Tooltip.Content)

The compound pattern helps maintain tree-shaking while providing a more ergonomic API.

### Using Hooks

```tsx
import { useToggle } from "@rhinolabs/hooks";

function App() {
  const [isOpen, toggle] = useToggle(false);

  return (
    <div>
      <button onClick={toggle}>
        {isOpen ? "Close" : "Open"}
      </button>
    </div>
  );
}
```
